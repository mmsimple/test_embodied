# 入职前小项目实战路线

你已经有测试基础，所以这里不再从测试概念开始，而是按可交付的小项目推进。

## 学习节奏

建议每个项目按 4 步完成：

1. 读懂需求：明确输入、输出、判定标准。
2. 跑通示例：先让现成脚本跑起来。
3. 修改功能：增加一个小需求，验证自己真的理解。
4. 总结成作品：保留 README、脚本、样例数据和结果报告。

## 项目顺序

### 项目 1：机器人日志分析脚本

目标：

- 从日志文件中统计 `ERROR`、`WARN`、`EXCEPTION` 等问题数量
- 提取高频错误信息
- 统计不同模块的问题分布
- 生成 Markdown 测试分析报告

为什么先做它：

- 入职后很快会接触日志
- 不依赖 ROS 或机器人硬件
- 能直接体现测试工程师的问题定位能力

目录：

```text
project_01_log_analyzer/
```

学习重点：

- Python 文件读取
- 正则表达式
- 日志结构化
- 问题分类
- Markdown 报告生成

### 项目 2：ROS2 小乌龟练习

目标：

- 启动 `turtlesim`
- 查看 topic、node、service
- 控制小乌龟运动
- 录制和回放 rosbag

学习重点：

- ROS2 基础命令
- 机器人数据流
- bag 数据复现问题

### 项目 3：机器人避障测试用例

目标：

- 设计静态障碍物、动态障碍物、窄通道、突然出现障碍物等测试场景
- 给出清晰的判定标准
- 输出一份可执行测试用例文档

学习重点：

- 场景设计
- 边界条件
- 安全测试
- 可量化验收标准

### 项目 4：自动化测试报告模板

目标：

- 根据测试结果生成 Markdown 或 HTML 报告
- 包含版本、环境、通过率、失败项、日志链接、结论

学习重点：

- 测试结果结构化
- 报告自动生成
- 测试闭环

### 项目 5：RViz 可视化练习

目标：

- 查看机器人模型
- 查看 TF 坐标树
- 查看点云、地图、路径

学习重点：

- 可视化调试
- 坐标系理解
- 传感器数据理解

## 当前开始

先从 `project_01_log_analyzer` 开始。

完成标准：

- 能运行脚本分析样例日志
- 能生成 Markdown 报告
- 能解释报告中的每个统计项
- 能自己新增一个统计规则
